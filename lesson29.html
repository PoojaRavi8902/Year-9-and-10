Lesson 29 – Test Data

Lesson Objectives

  • Understand the purpose of test data
  • Learn different types of test data
  • Use test data to check the correctness of algorithms

What is Test Data?

Test data refers to input values used to test whether a program or algorithm works correctly.

Programmers use test data to ensure that the program produces the correct output for different types of input.

Types of Test Data

There are three common types of test data:

  • Normal data
  • Boundary data
  • Invalid data

Normal Data

Normal data is valid input that falls within the expected range. The program should accept the data and produce the correct output.

Example:

If marks must be between 0 and 100, the value 75 is normal data.

Boundary Data

Boundary data is data that lies at the extreme limits of the valid range.

Example:

If marks must be between 0 and 100, the boundary values are:

  • 0
  • 100

Testing boundary values helps ensure the program handles edge cases correctly.

Invalid Data

Invalid data is input that falls outside the acceptable range. The program should reject this data.

Example:

  • -5
  • 120

Example Table of Test Data

Input Type of Test Data
75 Normal
0 Boundary
100 Boundary
-5 Invalid
120 Invalid

Why Test Data is Important

  • Ensures programs work correctly
  • Helps detect errors in algorithms
  • Improves reliability of software

Practice Questions

  1. What is test data?
  2. Explain the difference between normal data and boundary data.
  3. Give one example of invalid data.

Exam Style Question

A program accepts marks between 0 and 100. Provide examples of normal, boundary, and invalid test data.
