IGCSE Computer Science

Algorithm Design & Problem Solving (Exam-Friendly Notes)

Learning Objectives

  • Understand the stages of the Program Development Life Cycle (PDLC)
  • Explain computer systems and sub-systems
  • Apply abstraction and decomposition
  • Use flowcharts, pseudocode and structure diagrams
  • Explain the purpose of an algorithm
  • Use conditional statements and iteration
  • Identify and correct errors in algorithms

1. Program Development Life Cycle (PDLC)

Definition: The Program Development Life Cycle describes the stages involved in developing a computer program from the initial idea to a working solution.

Stages of PDLC

  • Analysis
  • Design
  • Coding
  • Testing
  • Maintenance
Exam Tip: You must be able to describe what happens in each stage, not just list them.

1.1 Analysis

Before solving a problem, it must be clearly defined. This produces a requirements specification.

Abstraction

Abstraction removes unnecessary detail and keeps only what is essential.

Decomposition

Decomposition breaks a complex problem into smaller, manageable parts.

Remember: Abstraction = remove detail Decomposition = break into parts

1.2 Design

The design stage shows how the program will work before coding begins.

  • Structure diagrams
  • Flowcharts
  • Pseudocode

1.3 Coding and Iterative Testing

The program is written in a suitable programming language. Each module is tested and corrected repeatedly until it works correctly.

1.4 Testing

The complete program is tested using different sets of test data to ensure all parts work together.

2. Computer Systems and Sub-Systems

Definition: A computer system consists of hardware, software, data, communications and people.

Each system can be divided into sub-systems until each part performs a single task.

2.1 Input – Process – Output (IPO)

Component Description
Input Data entered into the system
Process Actions performed on the data
Output Information produced
Storage Data saved for future use

3. Algorithms

Definition: An algorithm is a set of step-by-step instructions used to solve a problem.

Pseudocode Example

Price ← 10
Tax ← Price * 0.12
FinalPrice ← Price + Tax

4. Selection (IF and CASE)

IF Age < 18
  THEN OUTPUT "Child"
  ELSE OUTPUT "Adult"
ENDIF
  
CASE OF Day
  1 : OUTPUT "Monday"
  2 : OUTPUT "Tuesday"
  OTHERWISE OUTPUT "Invalid"
ENDCASE

5. Iteration (Loops)

FOR Loop

FOR Counter ← 1 TO 10
  OUTPUT "*"
NEXT

REPEAT UNTIL Loop

REPEAT
  INPUT Mark
UNTIL Mark = -1

WHILE Loop

WHILE Mark <> -1 DO
  INPUT Mark
ENDWHILE

Exam Success Tips

  • Use correct terminology
  • Indent pseudocode clearly
  • Label decision branches in flowcharts
  • Explain both purpose and process
