Lesson 22 – Trace Tables

Lesson Objectives

  • Understand what trace tables are
  • Learn how to trace algorithms step by step
  • Track changes in variable values during program execution

What is a Trace Table?

A trace table is a tool used to track how the values of variables change during the execution of an algorithm.

It helps programmers and students understand how an algorithm works step by step.

Trace tables are commonly used to test algorithms and identify logical errors.

Example Algorithm

total ← 0

FOR i ← 1 TO 3
    total ← total + i
NEXT i

OUTPUT total

Trace Table

i total
1 1
2 3
3 6

The trace table shows how the value of total changes during each iteration of the loop.

Steps for Creating a Trace Table

  1. Identify all variables used in the algorithm.
  2. Create columns for each variable.
  3. Execute the algorithm step by step.
  4. Record the values of variables after each step.

Example with an Array

total ← 0

FOR i ← 1 TO 3
    total ← total + marks[i]
NEXT i

Trace Table

i marks[i] total
1 70 70
2 80 150
3 60 210

Why Trace Tables Are Important

  • Help understand how algorithms work
  • Help detect logical errors
  • Useful for testing and debugging programs

Practice Questions
