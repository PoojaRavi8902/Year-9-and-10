🧠 3.1.1 — The Central Processing Unit (CPU)

The CPU (microprocessor) is the brain of the computer. It executes instructions and processes data. Modern CPUs are single microchips containing the control unit, arithmetic/logic unit and registers.

Hover over items during class. Use the CPU diagram to point out CU, ALU, Registers and how buses connect to memory.

Teacher prompt: Ask students: "Which register stores the address of the next instruction?" (Answer: PC)

🧩 3.1.2 — Von Neumann Architecture

Von Neumann introduced the stored-program concept: programs and data share the same memory, the CPU fetches instructions sequentially and executes them. This architecture uses the Address, Data and Control buses to coordinate actions.

Tip: emphasize the stored-program idea for exam answers — they expect you to mention programs and data are stored together in memory.

🔌 Registers, Buses & Memory

Registers are tiny storage inside the CPU used by the Fetch–Decode–Execute cycle. Buses are collections of wires used to transfer addresses, data and control signals.

  • PC — Program Counter: address of next instruction.
  • MAR — Memory Address Register: holds address being accessed.
  • MDR — Memory Data Register: holds data read from/written to memory.
  • CIR — Current Instruction Register: holds current instruction.
  • ACC — Accumulator: stores intermediate ALU results.

⏱️ Cores, Cache & Clock Speed

Factors affecting CPU performance:

  • Clock speed — number of cycles per second (GHz). Faster clock → more cycles per second but not always faster overall (bottlenecks exist).
  • Cache — tiny, extremely fast memory inside the CPU storing frequently-used data.
  • Cores — multiple cores allow parallelism (dual-core, quad-core). Communication overhead reduces perfect scaling.
Exam tip: When asked to explain performance, mention bus width, cache and cores — not just clock speed.

🔤 Instruction Set

An instruction is made of an opcode (what to do) and an operand (which data/address to use). The whole set of opcodes a CPU understands is its instruction set (e.g., X86).

Note: instruction sets are low-level machine instructions — they are not the same as high-level programming languages.

⚙️ Embedded Systems

Embedded systems are hardware+software solutions designed to perform dedicated tasks. They range from tiny microcontrollers to complex SoCs.

TypeDescription
MicrocontrollerCPU + RAM + ROM + peripherals on one chip — e.g., simple appliances
MicroprocessorCPU only — needs external memory and I/O
System on Chip (SoC)Complete system on a chip (CPU, memory, I/O)

Examples: cars (engine control, ABS), set-top boxes, smart thermostats, washing machines.
