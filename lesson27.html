Lesson 27 – Validation

Lesson Objectives

  • Understand the concept of validation
  • Learn how validation checks input data
  • Design algorithms that prevent invalid data entry

What is Validation?

Validation is the process of checking whether input data is sensible, reasonable, and acceptable before it is processed by a program.

Validation helps prevent incorrect or invalid data from entering a system.

Example

Suppose a program asks the user to enter their age. The age cannot be negative. 

INPUT age

IF age < 0 THEN
    OUTPUT "Invalid age"
ENDIF

Common Types of Validation

  • Range check – Ensures the value is within a specific range.
  • Length check – Ensures data has the correct number of characters.
  • Format check – Ensures data follows a specific pattern.
  • Presence check – Ensures that data is not left empty.

Example – Range Check

Marks must be between 0 and 100. 

INPUT mark

IF mark < 0 OR mark > 100 THEN
    OUTPUT "Invalid mark"
ENDIF

Example – Presence Check

INPUT name

IF name = "" THEN
    OUTPUT "Name cannot be empty"
ENDIF

Example – Using Validation with Loops

Programs often repeat input until valid data is entered. 

REPEAT
    INPUT age
UNTIL age >= 0

Why Validation is Important

  • Prevents incorrect data entry
  • Improves reliability of programs
  • Ensures accurate results

Practice Questions

  1. What is validation?
  2. Explain the purpose of a range check.
  3. Write pseudocode that ensures a number entered is positive.

Exam Style Question

Write pseudocode that asks the user to enter a mark between 0 and 100. The program must continue asking until a valid mark is entered.
