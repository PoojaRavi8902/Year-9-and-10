Lesson 28 – Verification

Lesson Objectives

  • Understand the concept of verification
  • Learn how verification ensures data accuracy
  • Differentiate between validation and verification

What is Verification?

Verification is the process of checking whether data has been entered correctly.

It ensures that the data entered into a system matches the original data.

Verification does not check whether data is sensible. Instead, it checks whether the data was entered correctly.

Example

When creating an online account, users are often asked to enter their password twice.

The system compares the two entries to verify that they are identical.

Example Algorithm

INPUT password
INPUT confirmPassword

IF password = confirmPassword THEN
    OUTPUT "Password verified"
ELSE
    OUTPUT "Passwords do not match"
ENDIF

Common Verification Methods

  • Double entry – Data is entered twice and compared.
  • Visual check – Data is visually checked against the original source.

Example – Double Entry

INPUT email
INPUT confirmEmail

IF email = confirmEmail THEN
    OUTPUT "Email verified"
ELSE
    OUTPUT "Emails do not match"
ENDIF

Difference Between Validation and Verification

Validation Verification
Checks if data is sensible Checks if data is entered correctly
Prevents invalid data Prevents data entry errors
Example: age cannot be negative Example: password entered twice

Why Verification is Important

  • Ensures data accuracy
  • Reduces input errors
  • Improves reliability of systems

Practice Questions

  1. What is verification?
  2. Explain one method of verification.
  3. Explain one difference between validation and verification.

Exam Style Question

Explain the difference between validation and verification with examples.
