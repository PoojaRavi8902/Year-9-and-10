Lesson 3 – Decomposition

Lesson Objectives

  • Understand the meaning of decomposition in computational problem solving
  • Learn how complex problems can be broken into smaller sub-problems
  • Understand how decomposition helps programmers design efficient algorithms

What is Decomposition?

Decomposition is the process of breaking a large and complex problem into smaller and more manageable parts. Each smaller part can then be solved individually.

In computer science, complex systems and programs are rarely created as a single large solution. Instead, programmers divide the problem into smaller tasks and solve them step by step.

This technique helps programmers organise their thinking and design clear and logical algorithms.

Why is Decomposition Important?

Decomposition provides several advantages in problem solving:

  • Complex problems become easier to understand
  • Each smaller task can be solved independently
  • Errors are easier to identify and correct
  • Programs become easier to design, test, and maintain
  • Different programmers can work on different parts of a system

Real-World Example

Consider the problem of organising a school sports day. Instead of handling everything at once, the task can be divided into smaller activities:

  • Registering participants
  • Scheduling events
  • Preparing equipment
  • Recording results
  • Announcing winners

Each task can be managed separately, making the overall organisation easier.

Example in Computing

Suppose we want to design a program that manages student examination results. The problem can be decomposed into smaller tasks:

  • Input student details
  • Input examination marks
  • Calculate total marks
  • Calculate average marks
  • Determine grade
  • Display the results

Each task can be designed as a separate part of the program.

Algorithm Example

Consider a program that calculates the average marks of a student. Using decomposition, the solution can be divided into steps: 

Step 1: Input marks for three subjects
Step 2: Calculate the total marks
Step 3: Calculate the average
Step 4: Display the average

Breaking the solution into these smaller steps makes the algorithm easier to design and understand.

Another Example

Design a program to calculate the area of a circle.

Decomposition of the Problem

  • Input the radius of the circle
  • Calculate the area using the formula
  • Display the result
INPUT radius
area ← 3.142 × radius × radius
OUTPUT area

Advantages of Decomposition in Programming

  • Improves problem solving efficiency
  • Helps design structured programs
  • Reduces complexity in large software systems
  • Makes programs easier to debug and maintain

Practice Questions

  1. Define the term decomposition.
  2. Explain why decomposition is useful when solving complex problems.
  3. Break down the task of creating a program that calculates the area of a rectangle.

Exam Style Question

A programmer is designing a system for a library. Explain how decomposition could help when designing the system.
