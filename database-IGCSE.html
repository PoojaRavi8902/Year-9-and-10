Quick overview (one-line cheatsheet)

Database = structured collection of data. Single-table database = one table only. A table = fields (columns) + records (rows). Primary key = unique identifier. SQL = use SELECT, FROM, WHERE, ORDER BY, SUM, COUNT to query.

Core concepts (student-friendly)

Table / Record / Field

Table: collection of related records (e.g., PATIENT).
Record: one row (data about one patient).
Field: one column (e.g., FirstName).

Primary key

Unique field (no duplicates, not NULL). Example: HospitalNumber or ISBN. Use short text or integer; ensure consistent format.

Validation vs Verification

Validation = automated checks (range, presence, format, length).
Verification = manual double-check (e.g., check an address is correct).

Basic data types (syllabus names)

  • text / alphanumeric — many characters (Access: Short Text / Long Text)
  • character — single character (Access: Short Text with length 1)
  • Boolean — True/False (Access: Yes/No)
  • integer — whole numbers (Access: Number / Whole number)
  • real — decimals (Access: Number / Decimal)
  • date/time — dates and times (Access: Date/Time)

Choose types so values display correctly and allow appropriate operations (SUM on numeric fields only, date comparisons on date fields).

How the table was created (SQL DDL)

CREATE TABLE PATIENT (
  HospitalNumber VARCHAR(8) PRIMARY KEY,
  FirstName VARCHAR(30),
  FamilyName VARCHAR(30),
  DateOfAdmission DATE,
  Consultant VARCHAR(30),
  WardNumber INTEGER,
  BedNumber INTEGER
);

Validation rules (examples): WardNumber BETWEEN 1 AND 10; BedNumber BETWEEN 1 AND 8; HospitalNumber format HN###### using a format check.

Exam-style questions — model answers (detailed)

Question 1 — CAR table (seat & paint combinations)

Part a i Number of records: 16 (assuming table shows 16 rows of different codes given the array in the question).
Part a ii Number of fields: 3 (Code, SeatMaterial, PaintColour) depending on how question presents — exam answer must match what is visible; if a header row shows Code, Cloth/Leather, PaintColours then count accordingly.

Part a iii Is any field suitable as primary key? Code is suitable because each code is unique for a combination; PaintColour or SeatColour are not unique.

Part b The SQL given is incorrect as written. The intention: list Codes where SeatColour = 'Red' AND PaintColour is either 'White' OR 'Silver'. Correct SQL: 

SELECT Code
FROM CAR
WHERE SeatColour = 'Red' AND (PaintColour = 'White' OR PaintColour = 'Silver');
Output: rows where SeatColour 'Red' and PaintColour 'White' or 'Silver' — show matching Code values from table.

Part c Customer wants leather seats and silver or grey paint: correct SQL: 

SELECT *
FROM CAR
WHERE SeatMaterial = 'Leather' AND (PaintColour = 'Silver' OR PaintColour = 'Grey');

Question 2 — CYCLE table (stock records)

a i Data types (recommended):

  • ModelNumber: TEXT (VARCHAR) — allows letters and digits (e.g., BY00007)
  • Description: TEXT
  • Colour: TEXT
  • WheelSize: INTEGER (mm — whole number)
  • Price: REAL (decimal)
  • StillManufactured: BOOLEAN (Yes/No)
  • NumberInStock: INTEGER

a ii Validation rules:

  • ModelNumber: presence check and format check (e.g., starts with BY and followed by 5 digits).
  • Price: must be >= 0 and numeric (range check, data type check).
  • NumberInStock: whole number >= 0 (integer range check).

a iii Suitable primary key: ModelNumber — unique identifier for each model (reason: unique, fixed for each model).

b Explain SQL commands with examples:

  • SELECT — choose fields to show. Example: SELECT ModelNumber, Description, Price FROM CYCLE;
  • FROM — specifies table. Example: FROM CYCLE
  • WHERE — filter records. Example: WHERE WheelSize = 700
  • SUM — add numeric field values. Example: SELECT SUM(NumberInStock) FROM CYCLE; returns total items in stock.
  • ORDER BY — sort results. Example: ORDER BY Price DESC (highest price first).

Exam strategy & common pitfalls

  • Always specify field names and table names exactly as given (case-insensitive for most SQL but match exam wording).
  • When writing string/date values in WHERE, enclose in single quotes (e.g., 'Mr Smith').
  • Use parentheses when combining AND/OR conditions to avoid logic errors.
  • Remember SUM only works on numeric fields; COUNT(*) counts rows.
  • When asked to give outputs, list values/rows in the order that the SQL would produce them.

Quick reference: SQL syntax

-- Select specific fields
SELECT Field1, Field2
FROM TableName
WHERE Condition
ORDER BY Field1 DESC;

-- Aggregates
SELECT SUM(Field) FROM TableName;
SELECT COUNT(*) FROM TableName;

Validation checklist (use in Access)

  • Presence (NOT NULL)
  • Type (must be integer / date / text)
  • Range (e.g., 1–10)
  • Length (exact characters or max length)
  • Format (e.g., HN\d{6})
