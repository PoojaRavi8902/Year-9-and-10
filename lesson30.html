Lesson 30 – Debugging

Lesson Objectives

  • Understand the meaning of debugging
  • Identify different types of programming errors
  • Learn how programmers find and fix errors

What is Debugging?

Debugging is the process of finding and correcting errors in a program or algorithm.

When a program does not work correctly, programmers analyse the code to identify the problem and fix it.

Debugging is an essential part of software development because even experienced programmers make mistakes.

Types of Errors

There are three common types of errors in programs:

  • Syntax errors
  • Logic errors
  • Runtime errors

Syntax Errors

A syntax error occurs when a program does not follow the rules of the programming language.

Example:

IF age >= 18
OUTPUT "Adult"

This statement is missing the keyword THEN, which causes a syntax error.

Logic Errors

A logic error occurs when the program runs but produces incorrect results.

Example:

average ← total / 2

If the program should divide by 3 instead of 2, the result will be incorrect.

Runtime Errors

Runtime errors occur while the program is running. These errors usually happen when the program attempts an illegal operation.

Example:

  • Dividing by zero
  • Accessing an array index that does not exist

Debugging Techniques

  • Use trace tables to track variable values
  • Check algorithm steps carefully
  • Test programs with different test data
  • Review the logic of the algorithm

Example Debugging Scenario

total ← 0

FOR i ← 1 TO 5
    total ← total + i
NEXT i

OUTPUT total

If the result is incorrect, a trace table can be used to track the values of total and i at each step.

Why Debugging is Important

  • Ensures programs work correctly
  • Improves software quality
  • Helps identify logical mistakes

Practice Questions

  1. What is debugging?
  2. Name three types of programming errors.
  3. Explain what a logic error is.

Exam Style Question

Describe two methods that programmers use to identify and correct errors in programs.
