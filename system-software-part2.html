High-level languages: Closer to human language, easy to read and portable. Examples: Python, C++, Java.

Low-level languages: Closer to machine code, hardware-specific. Example: Assembly.

FeatureHigh-LevelLow-Level
ReadabilityEasy for humansHard to read
PortabilityYesNo (hardware-specific)
Execution SpeedSlowerFaster
# High-level
sum = first + second

# Assembly
LDA FIRST
ADD SECOND
STO SUM
Think & Respond: Why do you think most programmers today prefer high-level languages?
They are easier to read, write, debug, and maintain, saving development time.

üîπ Phase 4 ‚Äì Mini Lecture 2: ‚ÄúCompilers, Interpreters & IDEs‚Äù

Click to read notes

üß© Translators

  • Compiler: Translates the whole program to machine code at once; creates an executable file.
  • Interpreter: Translates and executes line by line; stops when error occurs.
  • Assembler: Converts assembly to machine code.
FeatureCompilerInterpreter
TranslationEntire programOne line at a time
Execution SpeedFastSlow
DebuggingHarderEasier
ReusabilityExecutable fileNeeds re-interpretation

üß∞ IDE (Integrated Development Environment)

Provides tools for writing, testing, and debugging code.

  • Code editor
  • Debugger & runtime environment
  • Error diagnostics
  • Auto-completion & formatting
  • Translator (compiler/interpreter)
