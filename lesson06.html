Lesson 6 – Algorithm Representation

Lesson Objectives

  • Understand why algorithms need to be represented clearly
  • Learn the common methods used to represent algorithms
  • Understand the difference between pseudocode and flowcharts

Why Do We Represent Algorithms?

When solving computational problems, it is important to describe algorithms in a clear and structured way. This allows programmers and other developers to understand the logic of the solution before writing the actual program code.

Representing algorithms also helps in analysing, testing, and improving the solution. It ensures that the problem-solving steps are correct before implementation.

Common Methods of Representing Algorithms

There are several ways to represent algorithms in computer science. The two most common methods used in programming and examinations are:

  • Pseudocode
  • Flowcharts

Pseudocode

Pseudocode is a method of describing an algorithm using structured English that resembles programming language statements.

It does not follow strict programming language syntax, but it clearly describes the logical steps required to solve a problem.

Example pseudocode for calculating the area of a rectangle: 

INPUT length
INPUT width

area ← length × width

OUTPUT area

Pseudocode is widely used because it is easy to read and easy to convert into actual program code.

Flowcharts

A flowchart is a graphical representation of an algorithm. It uses different symbols connected by arrows to show the flow of steps in a process.

Flowcharts help programmers visualise the sequence of operations and decision points in a program.

Common flowchart symbols include:

  • Oval – Start or End
  • Parallelogram – Input or Output
  • Rectangle – Process
  • Diamond – Decision
  • Arrow – Direction of flow

Example Flow of an Algorithm

A simple algorithm for calculating the area of a rectangle using flowchart logic: 

Start
Input length
Input width
Calculate area
Display area
End

Advantages of Representing Algorithms

  • Improves understanding of the solution
  • Helps identify errors in logic
  • Makes communication between programmers easier
  • Provides a clear plan before coding

Practice Questions

  1. Why is it important to represent algorithms clearly?
  2. Name two common methods used to represent algorithms.
  3. Explain one advantage of using pseudocode.

Exam Style Question

Describe two methods that can be used to represent algorithms and explain how each method helps programmers.
