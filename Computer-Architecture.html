5.01 The von Neumann Model of a Computer System

Overview · Key features · Analogy · Exam tip

Overview

The von Neumann model describes a system where program instructions and data are stored together in memory and processed sequentially by a single processor (CPU). This architecture underpins most conventional computers.

Key features

  • Processor (CPU) — executes instructions.
  • Direct access to memory — CPU can read/write memory directly.
  • Stored program concept — data and instructions share memory; the program can be replaced.
  • Sequential execution — instructions executed in order unless changed by jumps/branches.

Diagram: Von Neumann model (simple)

Exam tip: The von Neumann architecture stores instructions and data in the same memory and uses the same bus — this can create a bottleneck where the CPU waits for memory transfers. Be prepared to explain or give consequences of the bottleneck in exam answers.

Analogy

Imagine a chef (CPU) using a cookbook (memory) where the recipes (instructions) and ingredients (data) are kept together. The chef reads a step, executes it, then reads the next step — sequential execution.

5.02 Central Processing Unit (CPU) Architecture

ALU · Control Unit · Clocks · Registers

CPU components (simplified)

The CPU typically contains the Control Unit (CU), the Arithmetic & Logic Unit (ALU), and a small set of registers. These components collaborate to fetch, decode and execute instructions.

Active components explained

  • ALU: performs arithmetic (add, subtract, multiply, divide) and logic (AND/OR/NOT, comparisons).
  • Control Unit: controls data flow, decodes instructions and coordinates CPU activities. It includes a clock used to synchronise operations.

Teacher note: emphasise the difference between the internal clock (CPU cycles) and the system clock (synchronises external transfers). In many exam questions, students must identify which clock governs which action.

Clocks and clock speed

The internal clock times operations inside the CPU; the system clock times those outside (memory, I/O). Clock speed is measured in Hz (e.g., 2 GHz = 2 billion cycles per second).

Classroom example: If a CPU is 2 GHz, one cycle is 0.5 nanoseconds. Ask students to calculate time for n cycles — helps them connect frequency and time.

Registers (detailed)

Registers are very fast storage elements within the CPU. They hold data, addresses, or instructions for immediate use.

RegisterAbbreviationFunction
Program CounterPCAddress of the next instruction to be executed.
Memory Address RegisterMARHolds the address of a memory location for read/write.
Memory Data Register / Memory Buffer RegisterMDR / MBRHolds data that has just been read from or is to be written to memory.
Current Instruction RegisterCIRStores the instruction currently being decoded/executed.
Index RegisterIXUsed for indexed addressing to modify addresses dynamically.
Status RegisterSRContains individual flag bits (Carry, Zero, Negative, Overflow).

Exam tip: be aware of abbreviation confusion: some texts use IR for Index Register or Current Instruction Register — in this page we use IX for index register and CIR for current instruction register.

Example (how registers work):

Instruction: ADD 400

  1. MAR ← 400
  2. MDR ← [address 400] (value fetched from memory)
  3. ALU adds MDR to Accumulator
  4. Result stored in Accumulator

5.03 The System Bus

Address bus · Data bus · Control bus · Examples

What is a bus?

A bus is a parallel set of wires that carries addresses, data or control signals between system components. It does not store data; it only transports it.

System bus components

BusDirectionPrimary function
Address busOne-way (CPU → memory/I/O)Carries addresses from MAR to memory or I/O controllers.
Data busTwo-wayCarries instructions, addresses or values between CPU, memory and I/O.
Control busTwo-wayCarries control signals, including timing signals from the system clock.

Diagram: System bus connecting CPU, memory and I/O

Example: Executing LOAD 500: CPU places 500 on the address bus, sends a READ on the control bus, and the memory value is returned on the data bus into the MDR.

5.04 Factors Contributing to System Performance

Clock speed · Cores · Cache · Word length · Bus width

Key factors

FactorWhat it meansEffect on performance
Clock speedNumber of cycles per secondHigher speed → faster execution (subject to other bottlenecks)
Number of coresIndependent processing units on chipMore cores → better parallelism for suitable tasks
Cache memoryVery fast memory on/near CPUMore/faster cache reduces wait for main memory
Word lengthNumber of bits handled as a unitLarger word → larger data processed per operation
Bus widthNumber of parallel lines in busWider bus → more data/addressable memory per transfer

Word length and addressing

A system's word length usually matches register sizes (16, 32, 64 bits). The address bus width determines how many memory locations can be addressed directly: e.g., 16-bit → 65,536 locations; 32-bit → ≈4.29 billion locations.

Teacher: ask students to show the calculation for 2^16 and 2^32; make them relate memory size to real RAM sizes (KB, MB, GB).
Example problem: Advertisement: "4 GB, 1 TB, 1.7 GHz" — ask students: which components are described? (Answer: 4 GB = main memory/RAM; 1 TB = mass storage; 1.7 GHz = CPU clock speed). Also ask: "Minimum time per cycle?" (1 / 1.7 GHz ≈ 0.588 ns)

Bus width vs word length

If data bus width < word length, multiple transfers are needed to send a single word — slowing the system. Designers aim to match bus width to word length where possible.

5.05 I/O Ports

Ports · USB · Specialised ports

Overview

Ports are interfaces through which I/O devices communicate with the system via device controllers. Ports may be internal (built-in devices) or external (peripherals).

USB (Universal Serial Bus)

  • Plug-and-play standard that automates device configuration.
  • Supports up to 127 devices on a host hierarchy.
  • Hot-swappable: devices can be attached/removed while powered on.
  • Version note: USB 3.2 supports high transfer rates (theoretical upto 20 Gbps with multi-lane configurations).
Class activity: choose two USB storage devices and list the USB version they use and their maximum transfer rates; compare to internal HDD/SSD speeds.

Specialised multimedia ports

PortUseCharacteristics
VGAMonitor connection (older)Analogue video only — no audio
HDMIModern monitors/TVsDigital video + audio; suitable for high-definition media

5.06 The Fetch–Execute (F–E) Cycle

Fetch · Decode · Execute · Flowchart · RTN

Flowchart (simplified)

The cycle repeats continuously while a program is running. Interrupts are checked periodically and handled between cycles.

Detailed fetch stage (register transfer notation)

When a program is running, the PC contains the address of the next instruction. In the fetch phase the following RTN describes the transfers:

MAR ← [PC]
PC ← [PC] + 1 ; MDR ← [[MAR]]
CIR ← [MDR]

Notes:

  • [X] means contents of register X.
  • separates simultaneous transfers.
  • [[MAR]] means memory content at the address stored in MAR.

Teacher: demonstrate RTN on the board step-by-step and ask students to annotate which lines depend on the system clock (hint: memory transfer and PC increment often align with different parts of the clock cycle).

5.07 Register Transfer Notation (RTN)

Notation · Meaning · Examples

RTN is used to express transfers between registers and memory compactly. Common symbols used on this page:

  • [X] — contents of register X.
  • ← — transfer into (destination is on the left).
  • [[MAR]] — memory contents at address stored in MAR.
  • ; — separates simultaneous operations.
Example RTN (fetch):
MAR ← [PC]
PC ← [PC] + 1 ; MDR ← [[MAR]]
CIR ← [MDR]

Students may be asked to write RTN for micro-operations or to label which transfers occur simultaneously — practise with small instruction sequences.

5.08 Interrupt Handling

Causes · Priorities · ISR · Steps

What is an interrupt?

An interrupt is a signal that temporarily halts a CPU's current execution to handle an event that requires immediate attention. Interrupts allow the CPU to respond to asynchronous events (e.g., I/O ready, errors, timers).

Common causes

  • Program error (e.g., division by zero)
  • Hardware fault (e.g., overheating, device fault)
  • I/O requests (e.g., keyboard/mouse data available)
  • Timer events (scheduling)
  • User interrupts (Ctrl+C)

How interrupts are handled

  1. At an interrupt point (end of F–E cycle), CPU finishes the current instruction.
  2. CPU saves PC and relevant registers to memory (stack or reserved area).
  3. Interrupt type is identified (often by bits in an interrupt register or vector table).
  4. PC is loaded with address of the Interrupt Service Routine (ISR).
  5. ISR executes. On completion, CPU checks for further interrupts.
  6. If none remain, saved register contents are restored and original program resumes.
Printer 'paper out' example: typing program is paused, printer ISR runs to alert user or queue job, then control returns to typing program.

Notes on priorities and nested interrupts

Interrupts may have priorities. Some systems allow nested interrupts (an ISR itself is interrupted by a higher-priority interrupt). Careful saving/restoring of state is essential to avoid corruption.

Classroom activities (ready-to-run)

  1. Whiteboard RTN practice: Give students short pseudo-instructions (LOAD 200, ADD 300, STORE 400) and ask them to write RTN and label simultaneous steps.
  2. Hands-on timings: Using stopwatch and sample program traces, ask students to compute elapsed time using different clock speeds and bus widths (use simplified assumptions).
  3. USB research: Assign two groups to compare USB versions for two flash drives and an internal HDD/SSD. Present speeds and discuss bottlenecks.
  4. Interrupt simulation: Role-play: assign students roles (CPU, memory, keyboard, printer). Generate interrupts and have CPU save/restore state manually to show the concept.
