Question 1 – Car Seat Materials & Colours

a (i) Number of records
Answer: 18 records.
Explanation: Each row in the CAR table represents one possible combination of code + material + seat colour + paint colour options.
a (ii) Number of fields
Answer: 9 fields.
Reason: The table contains: Code, Seat Material, Seat Colour, plus 6 paint‑colour fields.
a (iii) Primary key suitability
Answer: Field Code is suitable.
Reason: Each code (CB, LB, CC, LC, CG, LG, CR, LR…) uniquely identifies a combination line; no duplicates appear.
b SQL Output 
SELECT Code
FROM CAR
WHERE SeatColour = 'Red' AND ('White' OR 'silver');
Correct Interpretation:
The condition ('White' OR 'silver') is always TRUE in SQL.
So the query becomes:
WHERE SeatColour = 'Red' only.

Answer: All codes where SeatColour = Red.
c SQL completion 
SELECT *
FROM CAR
WHERE SeatMaterial = 'Leather' AND (PaintColour = 'Silver' OR PaintColour = 'Grey');

Question 2 – Bicycle Stock Database

a (i) Suitable data types (7 marks)
  • Model number – VARCHAR: alphanumeric values (e.g., BY00007).
  • Description – VARCHAR: text description.
  • Colour – VARCHAR: text.
  • Wheel size – INTEGER: whole number (e.g., 700).
  • Price – REAL: includes decimals (e.g., 309.50).
  • Manufactured? – BOOLEAN: Yes/No.
  • Number in stock – INTEGER: whole number.
a (ii) Validation needed (3 marks)
  • Model number – Format check (e.g., two letters + digits).
  • Price – Range check (e.g., price must be ≥ 0).
  • Number in stock – Range check (0 to sensible max).
a (iii) Primary key (2 marks)
Model number is ideal.
Reason: It is unique for each bicycle model.
b SQL command explanations (10 marks)
  • SELECT – chooses fields.
    Example: SELECT ModelNumber, Price
  • FROM – chooses the table.
    Example: FROM CYCLE
  • WHERE – filters records.
    Example: WHERE Colour = 'Gold and Black'
  • SUM – totals numerical fields.
    Example: SELECT SUM(NumberInStock)
  • ORDER BY – sorts results.
    Example: ORDER BY Price DESC

a Fields and Records

Records: 8 rows.
b Validation on Show Number
  • Format check – must follow pattern: two letters + three digits.
