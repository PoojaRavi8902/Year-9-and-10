What is a Database?

A database is an organised collection of data.

It is made up of either one or multiple tables which are made up of fields and records to organise how data is stored.

A database allows easy storage, retrieval, and management of information.

It is useful when working with large amounts of data, and databases are stored on secondary storage.

A database is often stored on remote servers so multiple users can access it simultaneously, making it useful for online systems.

Data can be sorted and searched efficiently using advanced structures.

They are more secure than text files.

Fields & Records

A field is one piece of information relating to one person, item, or object. In a database, a field is represented by a column.

A record is a collection of fields relating to one person, item, or object. A record is represented in a database by a row.

Text Files

A text file is useful when working with small amounts of data. Text files are stored on secondary storage and are read into a program when being used.

They are used to store information when the application is closed.

Each entry is stored on a new line or separated with a special identifier such as a comma (,).

It can be difficult in text files to know where a record begins and ends.

Validation

When a table is created, validation rules can be assigned to different fields.

A validation rule controls what data can be entered into that field.

There are different types of validation checks used to limit what data can be entered into each field.

TypeDescription
Length CheckChecks the number of characters entered. Example: phone numbers must be eleven characters long.
Format CheckEnsures data meets a specific format. Example: product code must be two letters followed by five numbers.
Range CheckEnsures a number is within a set range. Example: dog age must be between 0 and 40.
Presence CheckEnsures the field cannot be left blank.
Type CheckEnsures data matches a defined type. Example: date field should not accept text.
Check DigitsUsed to verify numbers such as credit card numbers.

Examiner Tips and Tricks

You will likely be presented with a database table and asked to identify the number of fields or records. Remember:

  • A record is a row.
  • A field is a column.

Data Types

A data type defines the kind of data a field can hold.

  • Integer – whole number
  • Real – decimal number
  • Text/Alphanumeric – text
  • Character – single character
  • Date/Time
  • Boolean – true/false

Example Table: Cars

  • car_id: integer
  • make: text/alphanumeric
  • model: text/alphanumeric
