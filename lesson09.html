Lesson 9 – Logical and Relational Operators

Lesson Objectives

  • Understand relational operators used in algorithms
  • Understand logical operators used in decision making
  • Learn how conditions are evaluated in algorithms
  • Apply logical expressions in pseudocode

Relational Operators

Relational operators are used to compare two values. The result of a comparison is either TRUE or FALSE.

These operators are commonly used in conditions within algorithms.

Operator Meaning Example
= Equal to a = b
< Less than a < b
> Greater than a > b
<= Less than or equal to a <= b
>= Greater than or equal to a >= b
<> Not equal to a <> b

Example of Relational Operators

INPUT age

IF age >= 18 THEN
    OUTPUT "Adult"
ENDIF

The condition age >= 18 is evaluated as either TRUE or FALSE.

Logical Operators

Logical operators are used when multiple conditions need to be evaluated together.

Operator Description
AND Both conditions must be TRUE
OR At least one condition must be TRUE
NOT Reverses the result of a condition

Example Using AND

INPUT age
INPUT citizenship

IF age >= 18 AND citizenship = "Yes" THEN
    OUTPUT "Eligible to vote"
ENDIF

Both conditions must be TRUE for the output to occur.

Example Using OR

INPUT temperature

IF temperature < 0 OR temperature > 35 THEN
    OUTPUT "Extreme temperature"
ENDIF

The output occurs if either condition is TRUE.

Example Using NOT

INPUT loggedIn

IF NOT loggedIn THEN
    OUTPUT "Please log in"
ENDIF

Importance of Logical and Relational Operators

  • Used to control decision making in algorithms
  • Allow programs to compare values
  • Essential for implementing conditional statements

Practice Questions

  1. What is the purpose of relational operators?
  2. Name three logical operators.
  3. Write a condition that checks if a number is greater than 10.

Exam Style Question

Write pseudocode to input a number and display "Positive" if the number is greater than zero.
