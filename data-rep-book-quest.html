Question 1 — Sound sampling, image resolution, colour depth, compression
1(a)(i) [1] What is meant by sampling resolution?
Answer: The sampling resolution (bit depth) is the number of bits used to record each audio sample — the number of bits used to represent the amplitude of each sampled point. Higher bit depth gives finer amplitude precision.
1(a)(ii) [2] Describe how sampling resolution affects accuracy of digitised sound.
  1. Higher sampling resolution (more bits per sample) reduces quantisation error and records amplitude more precisely — resulting in more accurate, less noisy digital sound. (1 mark)
  2. Lower sampling resolution increases rounding (quantisation) error so stored sound is less faithful to the original. (1 mark)
1(b)(i) [1] Explain the term image resolution.
Answer: Image resolution is the number of pixels in an image, usually given as width × height (for example, 1024 × 768). It indicates how many picture points (pixels) are used to form the image.
1(b)(ii) [1] 16-colour bitmap — how many bits per pixel?
Answer: 16 colours → needs log₂(16) = 4 bits per pixel. So 4 bits/pixel.
1(b)(iii) [3] Image size calculation
Given: width = 16 384 px, height = 512 px, saved as 256-colour bitmap → 8 bits/pixel.

Working:
Total pixels = 16 384 × 512 = 8 388 608 pixels.
Total bits = 8 388 608 × 8 = 67 108 864 bits.
Bytes = 67 108 864 ÷ 8 = 8 388 608 bytes.
Convert to GiB: 8 388 608 ÷ 1 073 741 824 = 0.0078125 GiB = 1/128 GiB.

Answer: 0.0078125 GiB.
1(b)(iv) [3] Describe file compression techniques the developer may use.
Answer (3 marks):
  • Lossless: RLE (run-length encoding), LZW, PNG compression. Reduce file size without losing any pixel data — use when exact pixel values are required (e.g., icons, sprites).
  • Lossy: JPEG — reduces file size by discarding some information (perceptually less important detail). Good for photographic backgrounds where small visual loss is acceptable.
  • Choice guidance: Use lossless when exact reproduction is required (game sprites, palette-sensitive images); use lossy (JPEG) when large size reduction is priority and small quality loss is acceptable (photographic textures/backgrounds).
Question 2 — Music sampling, compression and RLE
2(a) [3] Describe how sampling is used to record the music clips.
  1. Analogue sound waves are measured at regular time intervals (the sampling frequency, e.g. 44.1 kHz) producing discrete samples. (1 mark)
  2. Each sample’s amplitude is converted to a digital number using a chosen bit depth (sampling resolution) such as 16-bit. (1 mark)
  3. The sequence of samples is stored; during playback the samples recreate the waveform (digital-to-analogue). (1 mark)
2(b) [2] Which compression (lossy or lossless) should be used for email and why?
Answer: Use lossy compression (e.g. MP3, AAC) because it greatly reduces file size making email transfers practical while keeping acceptable audio quality for most uses. (2 marks) Note: If final mastering requires perfect fidelity, use lossless; for sending drafts by email, lossy is preferred.
2(c)(i) [1] What is meant by RLE?
Answer: Run-Length Encoding (RLE) stores consecutive repeated values as a single value plus a count, e.g. (0,5) for five consecutive zeros. (1 mark)
2(c)(ii) [3] Show how RLE would be used to produce a compressed file for the (exam) image.
Method & Example (3 marks):
  1. RLE records runs row-by-row. For each row, encode as pairs (value,count) where value is pixel code (given: grey=0, white=1). (1 mark)
  2. Example: if a row is 0 0 0 0 1 1 → RLE row encoding is (0,4),(1,2) or flat 0,4,1,2. (1 mark)
  3. For the actual exam image, list each row’s runs in sequence to form the compressed data. (If you paste the image grid here I will produce the exact RLE sequence.) (1 mark)
Question 3 — 8-bit register shifting
Given register (MSB → LSB): 0 0 1 1 0 1 1 0 (i.e. 00110110)
3(a) [1] Denary value of 00110110
Calculation: 32 + 16 + 4 + 2 = 54. Answer: 54.
3(b)(i) [1] Logical right shift 1 place — show result
Right shift inserts 0 at left: 00110110 → 00011011. Answer: 0 0 0 1 1 0 1 1.
3(b)(ii) [1] Denary value after right shift
00011011 = 16 + 8 + 2 + 1 = 27. Answer: 27.
3(c)(i) [1] Logical left shift two places — show result
Left shift 2: 00110110 << 2 → 11011000. Answer: 1 1 0 1 1 0 0 0.
3(c)(ii) [1] Effect on denary value and reason
A left shift by 2 multiplies the value by 2² = 4 (unless overflow occurs). Original 54 × 4 = 216. After shift register value = 11011000 = 216 — so multiplication by 4 succeeded with no overflow. (1 mark)

Question 5 — Bitmap image capacity on storage
Given: resolution = 1140 × 1080 pixels, colour depth = 24 bits
5(a) [1] Explain pixel.
Answer: A pixel (picture element) is the smallest addressable element of an image — a single point that holds a colour value.
5(b) [1] Explain colour depth.
Answer: Colour depth (bits per pixel) is the number of bits used to store each pixel’s colour. For 24-bit colour, typically 8 bits for each of red, green and blue.
5(c) [3] How many images on 32 GiB stick?
Pixels/image = 1140 × 1080 = 1 231 200.
Bits/image = 1 231 200 × 24 = 29 548 800 bits.
Bytes/image = 29 548 800 ÷ 8 = 3 693 600 bytes.
32 GiB = 32 × 2³⁰ = 34 359 738 368 bytes.
Number images = floor(34 359 738 368 ÷ 3 693 600) = 9 302 images.
Answer: 9 302 images.
5(d) [2] How to increase number of images stored?
Two possible measures (each 1 mark):
• Apply compression (e.g. JPEG) to reduce file size per image.
• Reduce resolution or reduce colour depth (e.g. from 24-bit to 8-bit).
Question 6 — Image capture and email compression
6(a) [4] How captured images are converted to digital files
Stepwise (4 marks):
1. The camera sensor (photosites) samples light intensity and colour as analogue voltages. (1 mark)
2. Each analogue value is converted to a digital number using an ADC (quantisation). (1 mark)
3. For colour, each pixel uses separate channels (RGB) or is reconstructed from Bayer data; pixels become digital triplets (e.g. 24-bit). (1 mark)
4. Pixel data are stored in a file format (JPEG, PNG, RAW), often with compression and metadata (EXIF). (1 mark)
6(b) [2] Which compression to email photos and why?
Answer: Use lossy compression (JPEG) because it achieves much smaller file sizes suitable for email, while maintaining acceptable photographic quality. (2 marks)

Question 8 — Memory stick, file reduction, RGB colour counts
8(a) [2] 64 GiB stick — how many 10 KiB photos?
64 GiB = 64 × 1 048 576 KiB = 67 108 864 KiB.
Number of 10 KiB photos = 67 108 864 ÷ 10 = 6 710 886 photos.
Answer: 6 710 886 photos.
8(b)(i) [1] One way to reduce file size:
Answer: Use lossy compression (convert to JPEG) or reduce image resolution.
8(b)(ii) [2] Two advantages of reducing image size:
1. Saves storage space (more images fit).
2. Faster upload/transfer times (e.g., email, web).
8(b)(iii) [1] One disadvantage:
Answer: Loss of image quality / irreversible detail loss if lossy compression is used.
8(c)(i) [1] Why 3 bytes per pixel in bitmap?
24-bit bitmap uses 8 bits for red, 8 bits for green, 8 bits for blue → 3 × 8 bits = 24 bits = 3 bytes per pixel.
8(c)(ii) [2] How many different colours?
Colours = 256 × 256 × 256 = 16 777 216 colours. (Answer)
