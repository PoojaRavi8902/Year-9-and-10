Lesson 2 – Problem Analysis

Lesson Objectives

  • Understand what problem analysis means in computing
  • Identify inputs, outputs, and processes of a problem
  • Understand the importance of analysing a problem before writing an algorithm

What is Problem Analysis?

Problem analysis is the first step in solving a computational problem. Before writing an algorithm or program, we must clearly understand the problem. This includes identifying what data is needed, what processing must occur, and what results should be produced.

If a problem is not analysed correctly, the final program may produce incorrect results.

Key Elements of Problem Analysis

Every computational problem usually contains three main parts:

1. Input

Input refers to the data that the program receives.

  • Numbers entered by the user
  • Data stored in a file
  • Sensor readings
INPUT length
INPUT width

2. Processing

Processing refers to the operations performed on the input data.

  • Calculations
  • Comparisons
  • Logical decisions
area ← length × width

3. Output

Output refers to the result produced by the program.

  • Displaying results on the screen
  • Printing a report
  • Saving results in a file
OUTPUT area

Example Problem Analysis

Problem: Calculate the average of three numbers.

Input

number1
number2
number3

Processing

sum ← number1 + number2 + number3
average ← sum / 3

Output

average

Why Problem Analysis is Important

  • Helps design correct algorithms
  • Reduces logical errors
  • Simplifies complex problems
  • Saves development time

Good programmers always analyse the problem carefully before writing code.

Practice Questions

  1. What is meant by problem analysis?
  2. Identify the inputs and outputs for a program that calculates the perimeter of a rectangle.
  3. Identify the processing step required to calculate the area of a triangle.

Exam Style Question

A program calculates the total cost of three items.

a) Identify the inputs.
